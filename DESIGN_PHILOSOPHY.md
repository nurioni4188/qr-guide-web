# 방문 민원 QR 길잡이 - 디자인 철학

## 설계 개요
공공기관 서비스의 신뢰성과 명확성을 기반으로, 모바일 사용자 중심의 직관적이고 효율적인 인터페이스를 구현합니다.

## 핵심 설계 원칙

### 1. 신뢰성과 명확성 (Trust & Clarity)
- 정부 서비스의 공식성 유지
- 정보 계층 구조의 명확한 표현
- 일관된 타이포그래피와 색상 체계

### 2. 모바일 우선 (Mobile-First)
- QR 스캔 후 즉시 접근 가능한 구조
- 터치 친화적 버튼과 간격
- 빠른 로딩과 명확한 네비게이션

### 3. 접근성 (Accessibility)
- 높은 명도 대비 (WCAG AA 이상)
- 명확한 포커스 상태
- 의미 있는 버튼과 라벨

### 4. 효율성 (Efficiency)
- 최소한의 클릭으로 정보 도달
- 명확한 행동 유도 (CTA)
- 불필요한 요소 제거

## 타이포그래피

### 폰트 선택
- **Display**: Noto Sans KR 900 (제목, 강조)
- **Heading**: Noto Sans KR 700 (섹션 제목)
- **Body**: Noto Sans KR 400 (본문)
- **Mono**: DM Mono 400 (코드, 메타 정보)

### 크기 체계
- H1: 26px / 900 weight
- H2: 18px / 700 weight
- H3: 15px / 700 weight
- Body: 13px / 400 weight
- Small: 12px / 400 weight
- Micro: 11px / 700 weight

## 레이아웃 원칙

### 간격 (Spacing)
- 기본 단위: 8px
- 섹션 간격: 32px (모바일), 40px (데스크톱)
- 컴포넌트 간격: 16px
- 내부 패딩: 20-24px

### 모서리 반경 (Border Radius)
- 카드: 14px
- 버튼: 8px
- 배지: 20px
- 입력: 8px

### 그림자 (Shadow)
- 기본: 0 4px 24px rgba(26,58,92,0.08)
- 호버: 0 8px 32px rgba(26,58,92,0.12)

## 상호작용 (Interaction)

### 애니메이션
- 전환 시간: 200-300ms
- 이징: ease-out (진입), ease-in-out (이동)
- 호버 효과: 미묘한 스케일 변화 (1.02x)

### 상태 표현
- **Done**: 초록색 배경 + 체크 아이콘
- **WIP**: 파랑색 배경 + 진행 중 표시
- **Todo**: 회색 배경 + 비활성 상태

## 반응형 디자인

### 브레이크포인트
- Mobile: < 600px
- Tablet: 600px - 1024px
- Desktop: > 1024px

### 모바일 최적화
- 터치 타겟 최소 44px × 44px
- 한 손 조작 가능한 레이아웃
- 큰 텍스트 (최소 14px)
- 충분한 여백

## 컴포넌트 가이드

### 카드
- 배경: 흰색 (#ffffff)
- 테두리: 1.5px solid #e8ecf2
- 패딩: 20-24px
- 호버: 그림자 강화

### 버튼
- Primary: 파랑 배경 + 흰 텍스트
- Secondary: 흰 배경 + 파랑 테두리
- 최소 높이: 40px
- 패딩: 10px 20px

### 배지
- 배경: 연한 파랑 (#eaf2fd)
- 텍스트: 진한 파랑 (#1a3a5c)
- 패딩: 4px 12px
- 폰트: 11px / 700 weight

## 접근성 체크리스트

- [ ] 색상 대비: WCAG AA 이상 (4.5:1 이상)
- [ ] 포커스 상태: 명확한 시각적 표시
- [ ] 키보드 네비게이션: 모든 상호작용 요소 접근 가능
- [ ] 스크린 리더: 의미 있는 라벨과 ARIA 속성
- [ ] 모바일: 터치 타겟 최소 44px
